# python-exceptions

## Description

This project is part of the Higher Level Programming curriculum. It covers
errors and exceptions in Python: how to catch, handle, and raise them
properly using `try` / `except` / `finally`.

## Learning Objectives

- Why Python programming is awesome
- What's the difference between errors and exceptions
- What are exceptions and how to use them
- When do we need to use exceptions
- How to correctly handle an exception
- What's the purpose of catching exceptions
- How to raise a builtin exception
- When do we need to implement a clean-up action after an exception

## Requirements

- Editors allowed: vi, vim, emacs
- All files are interpreted on Ubuntu 20.04 LTS using python3 (version 3.8.5)
- All files end with a new line
- The first line of all files is exactly `#!/usr/bin/python3`
- Code follows the `pycodestyle` style (version 2.7.*)
- All files are executable
- No modules may be imported unless a task explicitly allows it

## Tasks

| File | Description |
| --- | --- |
| `0-safe_print_list.py` | Prints x elements of a list, catching `IndexError` |
| `1-safe_print_integer.py` | Safely prints a value as an integer |
| `2-safe_print_list_integers.py` | Prints the first x integers of a list, skipping non-integers |
| `3-safe_print_division.py` | Divides 2 integers and prints the result, using `finally` |
| `4-list_division.py` | Divides two lists element by element, handling multiple exception types |
| `5-raise_exception.py` | Raises a `TypeError` exception |
| `6-raise_exception_msg.py` | Raises a `NameError` exception with a custom message |
